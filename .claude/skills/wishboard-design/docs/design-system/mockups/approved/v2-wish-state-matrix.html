State matrix — summary

15 состояний — один card skeleton

Все карточки шарят один скелет: thumb + prio-dot + title + desc + price/meta-row + trailing. Состояние меняет:

  • background tint — едва заметный диагональный gradient в tone-color (success-soft / accent-soft / warning-soft)
  • border color — tone-color с opacity 0.2-0.35
  • chip-set в meta-row — состояние сообщает себя текстом + иконкой
  • opacity — 0.55 для «не для тебя», 0.45 для «уже неактуально»
  • strike-through — для completed/deleted
  • progress-bar — для TTL (зелёный / оранжевый для expiring)

Правила

  • Один tone на state: success=reserved-by-me, accent=secret, warning=expiring, danger=conflict, muted=purchased/deleted
  • Privacy by default: guest не видит имени reserver'а. Owner видит что забронировано но не кем.
  • Derived states видимы: secret с ITEM_UPDATED / PUBLIC_RESERVED_BY_OTHER имеют свой уникальный визуал
  • Action inline: conflict предлагает «Promote» прямо в карточке, expiring — «Продлить»

Что лучше current prod

  • В current prod reserved/purchased/secret рендерятся через разные conditionals — каждая карточка чуть своего вида
  • Secret reservation сейчас имеет свой отдельный экран; здесь — unified card в потоке
  • Diff (ITEM_UPDATED) показан inline, не в отдельной модалке
  • TTL progress-bar — цветной signal вместо просто text-count
