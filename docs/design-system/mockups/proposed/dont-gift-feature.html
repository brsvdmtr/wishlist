UX-решения

  • Расположение: после 1-го желания — пользователь уже в контексте вишлиста, плашка выглядит как естественное улучшение
  • Вариант A лучше всего вписывается в существующий row-паттерн (как SettingsActionRow), минимум отвлечения
  • Вариант B более заметен, подходит для первого знакомства с фичей, есть кнопка "Скрыть"
  • Вариант C явно позиционирует как Pro-фичу, цветной акцент сверху привлекает внимание без навязчивости
  • Все варианты dismissable — по тапу "Скрыть" или через ×, запоминаем выбор per-wishlist

UX-решения

  • Гибридный формат: теги для быстрого выбора + текстовые пункты для конкретики + свободный комментарий
  • Live-превью: пользователь видит результат до сохранения — это мотивирует заполнить и убирает страх "а как это увидят"
  • Теги как toggle-chips: тап выбирает/снимает, красный фон выбранных визуально кодирует "нежелательное"
  • BottomSheet вместо отдельного экрана — не теряется контекст вишлиста, быстрее закрыть
  • Visibility toggle внизу — осознанный выбор после заполнения, не перед

UX-решения

  • Radio cards вместо segmented control — 3 варианта с описаниями не влезут в сегменты, а карточки дают место для пояснений
  • Статус-бейдж "✓ Заполнено · 3 категории" сразу показывает, что глобальный список уже настроен
  • Инлайн-форма раскрывается внутри выбранной карточки — не уводит на другой экран, контекст сохраняется
  • Упрощённая форма: в локальном режиме показываем только теги + комментарий (без кастомных пунктов) — достаточно для per-wishlist оверрайда. Полная форма — в настройках.
  • Отвергнуты: segmented control (мало места для текста), dropdown (скрывает варианты), list with checkmarks (не считывается как exclusive choice)

UX-решения

  • Расположение: между описанием и списком — дарящий видит контекст до того, как начнёт выбирать
  • Свёрнут по умолчанию: не перегружает экран, "5 пунктов" мотивирует раскрыть
  • Тон доброжелательный: иконка 🚫 в мягком розовом круге — не агрессивный запрет, а мягкая подсказка
  • Сепаратор "🎁 Список желаний" визуально отделяет блок от основного контента
  • Отвергнуты: расположение внизу (не увидят), всплывающий tooltip (раздражает), отдельная вкладка (overk kill)
Продуктовые заметки

Логика приоритета данных

  • 1. Wishlist mode = "hidden" → блок не показывается
  • 2. Wishlist mode = "local" + есть данные → показываем локальный список
  • 3. Wishlist mode = "global" → показываем глобальный список из UserProfile
  • 4. Глобальный список пуст → блок не отображается

Иерархия экранов (BottomSheet vs Inline)

  • Плашка в вишлисте: inline-block между карточками желаний
  • Paywall: BottomSheet (существующий ProUpsellSheet)
  • Настройка Pro: BottomSheet (85vh, scrollable)
  • Режим в ред. вишлиста: inline-block внутри формы редактирования
  • Публичный блок: inline collapsible card

Что потребуется для реализации

  • Schema: добавить dontGift* поля в модель Wishlist (сейчас только на UserProfile)
  • Schema: поле dontGiftMode на Wishlist: 'global' | 'local' | 'hidden' (default: 'global')
  • API: CRUD эндпоинты per-wishlist: GET/PUT /tg/wishlist/:id/dont-gift
  • API: обновить public endpoints — проверять wishlist mode, fallback на global
  • Frontend: баннер-компонент + dismiss state (localStorage per wishlist)
  • Frontend: radio cards в экране edit wishlist
  • Frontend: обновить гостевой рендеринг — поддержка local/global источника
  • Analytics: banner_shown/clicked, paywall_shown/cta_clicked, global/wishlist_created, mode_changed, public_block_shown
  • i18n: ~20 новых ключей для плашки, режимов, описаний
  • Paywall: новый upsell context 'dont_gift_banner' (отличать от 'dont_gift' в настройках)
  • God Mode: блок аналитики по фиче

Аналитические события

  • banner_shown_do_not_gift — плашка показана в вишлисте
  • banner_clicked_do_not_gift — тап по плашке
  • banner_dismissed_do_not_gift — плашка скрыта
  • paywall_shown_do_not_gift — paywall показан
  • paywall_cta_clicked_do_not_gift — CTA нажат
  • global_do_not_gift_created — глобальный список создан/обновлён
  • wishlist_do_not_gift_created — локальный список создан
  • wishlist_do_not_gift_mode_changed — режим изменён (global/local/hidden)
  • public_block_shown_do_not_gift — блок показан гостю
  • public_block_expanded_do_not_gift — гость раскрыл блок

Отвергнутые варианты

  • Segmented control для 3 режимов: не влезает текст описания, не сканируется за 1 секунду
  • Accordion list с чекмарками: визуально не считывается как exclusive choice (можно спутать с чеклистом)
  • Отдельный экран для настройки: слишком тяжело для ~30 сек заполнения, BottomSheet достаточно
  • Плашка сверху вишлиста: воспринимается как системное предупреждение, засоряет "витрину"
  • Плашка внизу: никто не доскроллит, низкая конверсия
  • Tabs в публичном блоке: overcomplicated для 1 секции, достаточно collapse/expand
  • Кастомные пункты в per-wishlist форме: избыточно, теги + комментарий покрывают 95% кейсов
