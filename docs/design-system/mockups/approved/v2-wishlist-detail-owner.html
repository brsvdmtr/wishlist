North Star — Wishlist · owner

Что показывает экран

Шапка

  • WL-emoji + title + sub: 🎂 эмодзи-якорь, 22/800 title с tight letter-spacing, sub — дата дедлайна «через 3 дня»
  • Meta-чипсы: режим видимости (🔗 link-only), политика комментариев (💬 все), don't-gift индикатор (🚫 3 пресета)
  • Stat-tile×3: желаний / забронировано (success-цвет) / куплено (accent). Быстрый glance-обзор без погружения в items.
  • Smart-reservation settings-row: текущий TTL «48 часов ›» — одна строка, не отдельный экран

Категории

  • Collapsible cat-header: chevron + имя + count + menu. Разворачивание вдоль, не в аккордеон
  • Empty category: dashed-border tile с «+», вместо «нет желаний» — делает пустую категорию полезной (точка входа для add)
  • Внизу — вход в «управление категориями» (modal с drag-reorder, rename, delete)

Item card (owner вид)

  • Thumb 56×56 с эмодзи + priority-dot в углу (маленький badge с эмодзи-эмоцией)
  • Title + desc — одной строкой с ellipsis
  • Meta-row: price в accent + shared-chip «🔗 в 2 вишлистах» + source chip
  • Trailing: menu-dot вверху (⋯), badge внизу («✓ забронировано» — success-chip)
  • Shared-wish: borderLight переходит в accent-soft border — subtle но читаемо

Sticky CTA

Primary-gradient «+ Добавить желание» с glow. Альтернативы (import from URL, категории) — в overflow меню (⋯) сверху.

Что лучше current prod

  • Stat-tiles дают мгновенный summary — сейчас это растворено
  • Smart-reservation TTL один клик — сейчас глубже в settings
  • Priority-dot на thumb вместо отдельного emoji-column — компактнее
  • Shared-wish badge интегрирован в meta-row, не отдельной строкой
  • Категории получают реальный empty state — точка добавления
