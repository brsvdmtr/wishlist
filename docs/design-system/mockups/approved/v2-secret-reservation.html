North Star — Secret reservations

Derived state machine как визуальный язык

Тайная бронь — уникальная фича WishBoard. У неё есть пять автоматических states (из SecretReservationDerivedState), и пользователь должен различать их с одного взгляда.

Визуальная кодировка

  • State-strip (левая граница): 3px цветная полоса — мгновенный signal тона state'а
  • Background tint: linear-gradient из card в tint-color с opacity 0.05 — едва заметно, но считывается
  • State-chip в meta-row: «✓ ACTIVE» / «⚠️ ITEM_UPDATED» / «⚡ ПУБЛИЧНО ЗАНЯЛИ» / «💳 ITEM_FULFILLED» / «🗑 ITEM_UNAVAILABLE»
  • Action-row: меняется под state (ACTIVE — нет; UPDATED — «Сравнить / Принять»; CONFLICT — «Отказаться / → Сделать публичной»)
  • 🔒 бейдж на thumb — универсальный маркер «это тайная бронь»

5 derived states

ACTIVE — всё хорошо, безопасно покупать
ITEM_UPDATED — владелец правил (цена / фото / описание). Показываем diff + «Сравнить / Принять»
PUBLIC_RESERVED_BY_OTHER — конфликт: кто-то публично забронировал. CTA «→ Сделать публичной» прямо в карточке.
ITEM_FULFILLED — владелец отметил купленым. Авто-возврат брони, никто не узнал.
ITEM_UNAVAILABLE — владелец удалил желание. Авто-возврат.

Hero-блок сверху

Brand gradient card с 🔒-декорацией. Объясняет что такое «тайная бронь» + summary «5 активных · 2 требуют внимания». Ensures даже новички понимают intent.

Filter chips

«изменилось 1», «конфликт 1» — chip-filters показывают что нужно внимание. Цвет chip = цвет state.

Что лучше current prod

  • State-strip даёт мгновенный визуальный signal вместо только chip
  • Diff показан inline (цена: 1 890 ₽ → 2 450 ₽) — не в отдельной модалке
  • Conflict имеет кнопку «→ Сделать публичной» прямо в карточке — 1 tap resolve
  • FULFILLED и UNAVAILABLE автомат-объяснения («никто не узнал», «авто-возврат»)
  • Hero hero-блок с чётким value-prop — сейчас фича зарыта глубоко
