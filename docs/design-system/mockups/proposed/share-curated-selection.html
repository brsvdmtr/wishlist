UX-решения

  • Вариант A (меню) — точка входа для сценария «хочу поделиться подборкой», открывает свой selection flow
  • Вариант B (bulk bar) — для сценария «уже выбрал желания, хочу отправить». Экономит один шаг
  • Оба варианта используются одновременно — это не либо/либо
  • PRO-бейдж в меню сигнализирует премиальность без блокировки UI
  • В bulk bar кнопка "Подборка" голубая — визуально отличается от destructive actions (красный, оранжевый)

UX-решения

  • Реиспользуется существующий multi-select паттерн (checkboxes 24x24, accent background)
  • Счётчик "Выбрано: N из M" — ясность, сколько всего и сколько выбрано

UX-решения

  • BottomSheet вместо полноэкранного — сохраняем контекст, быстро закрыть
  • Дефолт "Идеи подарка" — пользователю не надо думать, если не хочет
  • Мини-превью — подтверждение выбора, нет сомнений "а что я выбрал?"
  • TTL-предупреждение оранжевое — заметно, но не пугает
  • Snapshot-подсказка — объясняет иммутабельность без технических терминов
  • Лимит названия — 100 символов, счётчик справа
  • Пустое название → красная рамка, CTA disabled
  • Закрытие sheet → ничего не создаётся, безопасный выход

UX-решения

  • Success-first — сразу показываем результат и действия
  • Telegram share — основной CTA (голубой), как в существующем share flow
  • Копировать ссылку — для non-Telegram шеринга
  • "Отключить ссылку" — вторичное действие, мелкий текст, без акцента
  • TTL-бейдж напоминает о сроке жизни
  • Error state — понятный, с retry, без паники

UX-решения

  • Changelog — стандартный канал discovery, пользователи проверяют при red dot
  • Нудж на share-экране — контекстный: пользователь уже в sharing mindset, самый высококонверсионный момент

  • Отвергнуто: popup/modal при заходе в вишлист, floating badge на кнопке "Поделиться", push-уведомление
Продуктовые заметки

Полный UX-flow

  • 1. Пользователь в вишлисте → тапает "Управление" → "Поделиться подборкой"
  • 1b. Или: long press → bulk select → выбирает → тапает "Подборка" в action bar
  • 2. FREE → paywall. PRO → selection mode
  • 3. Выбирает 1+ желаний → "Продолжить"
  • 4. BottomSheet: название + превью + TTL notice → "Поделиться"
  • 5. Создание snapshot → success state → Telegram share / copy link
  • 6. Получатель открывает ссылку → light-подборка без бронирования
  • 7. После 45 дней → expired state

Edge cases — обработка в дизайне

  • 1 желание — валидно, CTA активен, мини-превью показывает 1 карточку
  • Много желаний (50+) — мини-превью скроллится, показывает первые 5 + "и ещё N"
  • Снял весь выбор — CTA деактивируется, счётчик "Выбрано: 0"
  • Пустое название — красная рамка, текст ошибки, CTA disabled
  • Длинное название (100+ символов) — обрезка ввода, счётчик символов
  • Закрыл sheet — ничего не создаётся, swipe-to-dismiss безопасен
  • Ошибка API — error state с retry и cancel
  • Удалённое желание — snapshot не зависит от оригинала, карточка остаётся
  • Изменённое желание — snapshot иммутабелен, старая версия сохраняется
  • Без фото — emoji-placeholder (существующий паттерн wish-thumb)
  • Без цены — строка цены скрывается
  • Длинное название желания — text-overflow: ellipsis (существующий паттерн)
  • После TTL — expired state, CTA на создание вишлиста
  • Повторный share — success state с copy/share остаётся доступным
  • Маленький вишлист (1-2 желания) — работает, можно выбрать 1

Что потребуется для реализации (следующий этап)

  • Schema: модель CuratedSelection (id, wishlistId, title, items JSON snapshot, shareToken, expiresAt, deactivatedAt, createdAt)
  • API: POST /tg/wishlists/:id/selections — создание, GET /public/selections/:token — публичный, DELETE — деактивация
  • TTL: expiresAt = createdAt + 45 days, cron cleanup опционально
  • Frontend: selection flow (reuse bulk-select), configure sheet, success state, public page
  • Share: Telegram deep link + clipboard
  • i18n: ~30 новых ключей
  • Analytics: selection_started, selection_created, selection_shared, selection_link_opened, selection_expired_viewed, paywall_shown_selection
  • Paywall: новый UpsellContext 'curated_selection'
  • Public page: /s/[token] — SSR light-version

Отвергнутые варианты

  • Отдельный FAB "Подборка" — перегружает интерфейс, конфликтует с "Добавить желание"
  • Подборка как тип вишлиста — противоречит требованию "не отдельный вишлист"
  • Drag-and-drop выбор — сложно на мобильном, multi-select проще
  • Бронирование в подборке — явно исключено по ТЗ, усложняет snapshot-модель
  • Auto-sync с вишлистом — противоречит snapshot-модели, создаёт путаницу
  • Popup при входе в вишлист — навязчиво, не premium feel
  • Бессрочная ссылка — не нужно, 45 дней оптимально для подборки
