Решение по навигации — выбран Вариант A ✅

«Мои списки» переехали с главной в отдельную вкладку. Выбран Вариант A (5 вкладок) — минимум слома привычки: свежие «Близкие» и «Брони» остаются. Реализовано: 🏠 Главная(лента) · 📋 Списки · 👥 Близкие · 🎁 Брони · 👤 Я.

Вариант A — 5 вкладок ✅ выбран

Минимум изменений для текущих юзеров: «Близкие» и «Брони» (только что выкатили) остаются. Добавляем только «Списки». Блок «Мои брони» в ленте — это шорткат-превью, а вкладка «Брони» — полный менеджер. Дублирования нет.

Покрытие acceptance criteria

✓ AC-1

События по близости с countdown. Phone ①: карточки отсортированы по близости, у каждого события — бейдж обратного отсчёта (сегодня / 3 дня / 18 дней).

✓ AC-2

Новое желание близкого → карточка с переходом. Phone ①: «Марина добавила 3 желания» + превью + CTA «Посмотреть» → её список.

✓ AC-3

Нет кругов → не пустота, а CTA. Phone ③: мост к P0.1, объяснение ценности + «Создать круг».

✓ AC-4

Тап по событию → сразу нужный список. CTA «Выбрать подарок» ведёт прямо в список именинника, без промежуточного экрана.

✓ Доп.

«Ты забронировал — не забудь». Phone ①: зелёная карточка брони + «Куплено» / «Детали». Surprise-invariant: бронь из CircleReservation, владельцу не видна.

✓ Доп.

«Мои брони» отдельным блоком + фильтр по кругам (Phone ②). «Цена упала» помечена P1.1 — покажем, когда подъедет трекинг цен.

PROPOSED · черновик для апрува. Источник токенов — packages/ui-tokens + colors_and_type.css (v2.1, тёмное стекло на mesh).
