D — Без подписок на профиль, только переходы
Не вводим новую сущность «подписка на профиль». Вместо этого делаем аватарки владельцев тапабельными везде, где они показываются — в карточке подписки, в шапке чужого вишлиста, в комментариях и т.д. → открывается публичный профиль.
1. На карточке подписки «Кошкин вишлист» тапаешь на аватарку/имя Екатерины → открывается её профиль (не вишлист).
Тап на остальную часть карточки — по-прежнему ведёт в сам вишлист.
2. Внутри чужого вишлиста в шапке тапаешь на аватар/имя владельца → профиль.
Это уже работает по сути (аватар виден), просто нужно сделать его тапабельным.
3. В комментариях, в списке бронирующих — везде где видно другого человека, тап по его имени/аватару → профиль.

Мой вывод

Если цель — монетизация через Showcase и соцграф, то твой вариант A (два блока) — самый правильный: он делает «профиль» полноценной первоклассной сущностью и стимулирует пользователей выкладывать Showcase, чтобы собирать подписчиков.

Если цель — сделать переходы в профиль сейчас, без новых фич, то D — самый дешёвый и безопасный: разблокирует навигацию за пару часов работы, без миграций и лишних экранов.

Я бы в итоге сделал D сейчас (тапабельные аватарки везде — это база, должно быть в любом случае), а A позже как отдельную фичу «фолловить человека», когда будет достаточно пользователей с опубликованным Showcase.
